# Client Example Resources

This directory contains the FHIR resources used to populate the Inferno Reference Server for US Quality Core client testing.

The files in this directory are mounted into the reference server container by [`docker-compose.background.yml`](/Users/Shared/Docker/qmcm/us-quality-core-test-kit/docker-compose.background.yml). When the reference server starts with an empty database, it loads these resources automatically.

If you add, remove, or update files here and want those changes to be reloaded, remove the existing Docker volume first:

```sh
docker volume rm us-quality-core-test-kit_fhir-pgdata
```

## Contents

- `us-quality-core-bundle.json`
  A transaction bundle containing the sample patient and related clinical data used by the client test suite.

- `SearchParameter-us-quality-core-*.json`
  Custom search parameter definitions needed for US Quality Core testing when the required behavior is not covered by the base FHIR specification or US Core.

- `example_client_v010.rb`
  An autogenerated example client script demonstrating requests against the US Quality Core client test setup.

## Notes

- These resources support the local reference server used for client-side Inferno testing.
- Changes in this directory do not take effect on an already initialized reference server unless the backing data volume is cleared.
